# Common Term Pitfalls

Quick-reference for commonly misused terms from the Microsoft Writing Style Guide A–Z word list.

## Individual terms

| Term | Guidance |
|---|---|
| **allow, allows** | Don't use *allows/enables/lets* to describe what Microsoft products make possible. Rewrite to focus on the customer. OK for security features that permit/deny access. |
| **app** | Use *app*, not *application*, for desktop and cloud apps. OK to use *application* for technical audiences. Don't use *application program*. |
| **back end** | Two words as a noun, hyphenate as an adjective (*back-end server*). Prefer a specific term like *server*, *database*, or *network*. |
| **blacklist** | Never use. Use *block list*. For email, use *blocked senders list*. |
| **click** | Avoid — specific to mouse. Use *select* instead. OK when describing mouse actions specifically. |
| **e.g.** | Don't use. Use *for example*, *such as*, or *like*. |
| **enable, enables** | Avoid. Don't use to describe what Microsoft makes possible (same rule as *allows*). Don't use to mean making a feature available — use *turn on*, *allow*, *add*. In UI, use *On/Off* not *Enable/Disable*. |
| **ensure, insure, assure** | *Ensure* = make sure/guarantee. *Insure* = provide insurance. *Assure* = state positively/make confident. |
| **execute** | Don't use. Use *run* instead. Even if the UI says "Execute", the action is *run*. Always *run* for macros and queries. |
| **gray, grayed out** | Don't use to describe unusable commands. Use *not available* or *isn't available*. Use *appears dimmed* only if you must describe appearance. |
| **he/she** | Don't use. See bias-free communication guidelines. |
| **hit** | Don't use for pressing a key. Use *select*. For web metrics, use *page view* or *page request*, not *hit*. |
| **i.e.** | Don't use. Use *that is* instead. |
| **initiate** | Don't use to mean *start a program or app*. |
| **launch** | Don't use to mean opening an app or form. Use *open*. |
| **leverage** | Don't use as a verb. Use *take advantage of*, *use*, or a more specific word. |
| **master/slave** | Don't use. Use *primary/replica*, *primary/secondary*, *principal/agent*, *controller/worker*. Don't use *primary/subordinate* as a synonym for *parent/child*. |
| **OK, okay** | Use *OK*. Don't use *okay*, *all right*, or *alright*. |
| **once** | Don't use as a synonym for *after*. Write "After you save the document…" not "Once you save…" |
| **open** | Use *open*, not *start* or *launch*, for programs and apps. Use *open* (not *opened*) for the open state. Don't use *open* for selecting a command or menu option. |
| **please** | Avoid except when the customer is inconvenienced or the app is at fault. |
| **right-click** | Hyphenate. Use only for mouse-specific instructions. For input-neutral instructions, use *select and hold (or right-click)*. |
| **run vs. execute** | Use *run*, not *execute*. Always *run* for operating systems, macros, and queries. |
| **select** | Use for marking text, objects, cells. Don't use *highlight* or *pick* as synonyms. |
| **should vs. must** | *Should* = recommended but optional. *Must* = required. Prefer imperative mood or *we recommend* over both. Don't use *should* for probability — use *might*. Don't use *Microsoft recommends* or *it is recommended*. |
| **simply** | Don't use to mean something is easy. Just state the action. |
| **start** | Don't use to describe opening a program or app — use *open*. Use *restart* for closing and reopening. |
| **terminate** | Don't use as a synonym for *close* or *exit*. OK in technical phrases like *null-terminated string* or *terminate a process*. |
| **that vs. which** | *That* = restrictive clause (essential, no comma). *Which* = nonrestrictive clause (parenthetical, preceded by comma). Include *that* even when the sentence seems clear without it. Use *who* for people, not *that* or *which*. |
| **uncheck** | Don't use. Use *clear* to describe clearing a checkbox. Also avoid *unmark* and *unselect*. |
| **user, end user** | Don't use *end user*. Avoid *user* — prefer *customer*, *person*, *people*, *you*. OK in developer content to distinguish developer from user. |
| **want** | Use instead of *wish* or *desire*. Don't use *want* when you mean *need* (a requirement). |
| **we** | Avoid making Microsoft the subject. OK for *we recommend* if it avoids awkward phrasing like *it's recommended*. Focus on the customer. |
| **whitelist** | Never use. Use *allow list*. For email, use *safe senders list*. |

## Bias-free and inclusive language

| Avoid | Use instead |
|---|---|
| blacklist | block list, blocked senders list |
| whitelist | allow list, safe senders list |
| master/slave | primary/replica, primary/secondary, controller/worker |
| he/she, he or she | they, the customer, the user (rewrite to avoid gendered pronouns) |
| black hat hacker | computer criminal, unauthorized user, malicious hacker |
| white hat hacker | computer security expert |

## Interaction verbs (input-neutral writing)

| Avoid | Use instead | Notes |
|---|---|---|
| click | select | *Click* is mouse-specific; *select* works across devices |
| click on | select | Never *click on* |
| double-click | (keep for mouse-specific content) | Hyphenate; for touch, use *double-tap* |
| right-click | select and hold (or right-click) | Use the parenthetical form for input-neutral instructions |
| hit (a key) | select | Never use *hit* for keyboard actions |
| press (a key) | select | Use *select* for key presses in general content |
| launch (an app) | open | Never *launch* for opening apps |
| start (an app) | open | Reserve *start* for other uses |
| execute | run | Always *run* for macros, queries, programs |
| initiate | start (or rewrite) | Don't use for starting programs |
| tap | (keep for touch-specific content) | Use *select* for input-neutral content |

## UI state and feature terms

| Avoid | Use instead |
|---|---|
| grayed out | not available, isn't available |
| (describing dimmed appearance) | appears dimmed |
| Enable/Disable (UI labels) | On/Off, show/hide, add/remove, allow/block |
| uncheck / unmark / unselect | clear (a checkbox) |

## Jargon and formality

| Avoid | Use instead |
|---|---|
| leverage (verb) | use, take advantage of |
| utilize | use |
| desire, wish | want |
| please (routine instructions) | (omit — just state the action) |
| simply, just | (omit — don't imply ease) |
| e.g. | for example, such as, like |
| i.e. | that is |

## Cloud and infrastructure terms

| Term | Guidance |
|---|---|
| the cloud | Don't capitalize *cloud* (except *the Microsoft Cloud*). Use *cloud* as adjective; prefer *cloud computing* or *cloud services* over "the cloud". |
| on-premises | Hyphenate in all positions. Always *premises* (plural). Never *on-premise*. |
| IaaS, PaaS, SaaS | Spell out on first mention. Don't capitalize as IAAS/PAAS/SAAS. Don't hyphenate as modifiers. Technical audiences only. |
| cloud native | Two words. Hyphenate before a noun (*cloud-native app*). Don't use *born in the cloud*. |
| serverless | One word, no hyphen. |

## Security terms

| Term | Guidance |
|---|---|
| hacker | Use *malicious hacker* for bad actors; *unauthorized user* if intent is unknown. OK in positive developer contexts (hackathons). |
| insecure | Don't use to mean *not secure*. |
| malicious code | Don't use. Use *malware* or *malicious software*. |
| bullet-proof | Don't use. |

## Keyboard and key terms

| Avoid | Use instead |
|---|---|
| hot key, fast key, quick key | keyboard shortcut |
| keypress | keystroke |
| Escape | Esc |
| press, depress, strike (a key) | select |
| hit (a key) | select |
