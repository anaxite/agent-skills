# Punctuation

> Distilled from the Microsoft Writing Style Guide — Punctuation section.
> The more punctuation you add, the more complex a sentence becomes. If a sentence has more than a comma or two plus end punctuation, consider rewriting.

## Apostrophes

### Use an apostrophe

- Use 's to form possessives of singular nouns, even those ending in *s*, *x*, or *z*
  - ✓ insider's guide, the box's contents, the CSS's flexibility, Berlioz's opera
- Add only an apostrophe to form possessives of plural nouns ending in *s*
  - ✓ users' passwords, the Joneses' computer
- Use to indicate a missing letter in a contraction
  - ✓ can't, don't, it's

### Don't use an apostrophe

- For the possessive of *it* → use *its*
  - ✓ Replace a formula with its calculated value.
- With possessive pronouns (yours, ours, theirs)
- To form the plural of a singular noun
  - ✗ Play your favorite game's → ✓ Play your favorite games
- Don't use possessive forms of Microsoft trademarks or product names

## Colons

- Avoid colons to introduce clarification mid-sentence — rephrase, use a comma, or use an em dash instead
- If you must use a colon mid-sentence, lowercase the word after it (unless proper noun or direct quotation)
- Don't use a colon at the end of titles, headings, or UI labels
- Capitalize the word after a colon in a title/heading that has a title–subtitle structure
  - ✓ Block party: Communities use Minecraft to create public spaces
- Don't use a colon before images, tables, or code samples — use a period
- Before lists:
  - Use a colon if the intro is a complete sentence that refers to the list ("the following," "as follows," a stated count)
  - Use a period if the intro sentence doesn't directly reference the list
  - Use a colon if the intro is a fragment and at least one list item completes it as a sentence
  - Use no punctuation if the fragment doesn't form a sentence with any item (consider making it a heading)
- In UI: don't use colons after headings, input labels, command names, or button/checkbox labels
- When referring to a UI element in docs, omit the colon even if the UI has one

## Commas

### Use a comma

- Before the conjunction in a list of three or more items (Oxford/serial comma)
  - ✓ Outlook includes Mail, Calendar, People, and Tasks.
- After an introductory phrase
  - ✓ With the Skype app, you can call any phone.
- To join independent clauses with a conjunction (*and, or, but, so*)
  - ✓ Select **Options**, and then select **Enable fast saves**.
- Between two or more pre-noun adjectives whose order could be reversed
  - ✓ Adjust the innovative, built-in Kickstand and Type Cover.
- Around the year in a full date within a sentence
  - ✓ …the February 4, 2015, issue…

### Don't use a comma

- To join independent clauses without a conjunction — use a semicolon
  - ✗ Select **Options**, then select **Enable fast saves.**
- Between verbs in a compound predicate (two verbs, one subject)
  - ✓ The program evaluates your system and then copies the files.
- Between month and year when no specific date is given

## Em dashes

- Use to set off a break in thought or parenthetical remark within a sentence
  - ✓ The information in your worksheet—numbers, formulas, and text—is stored in cells.
- Don't overuse — they affect readability if excessive
- Don't use spaces around em dashes
  - ✗ The information — numbers — is stored in cells.
- Don't use multiple hyphens in place of an em dash

## En dashes

- Use in ranges of numbers and dates
  - ✓ 2015–2017
- Use spaces around en dashes only in time stamps in UI or ranges with both times and dates
  - ✓ 2:15 PM – 4:45 PM
  - ✓ 2:15 PM 12/1/17 – 4:45 PM 4/1/18
- Use instead of a hyphen in compound modifiers containing an open compound
  - ✓ Windows 10–compatible products, AI Builder–supported features
- Don't use an en dash in place of a minus sign (accessibility concern)
- Don't use spaces around en dashes in number/date ranges

## Hyphens

### Predicate adjectives

- Don't hyphenate predicate adjectives (after a linking verb) unless the style guide specifically says to
  - ✓ The text is left aligned. The camera is built in.

### Noun modifiers

- Don't hyphenate *very* + modifier or adverb ending in *-ly* + modifier before a noun
  - ✓ very fast test, extremely stylized image, highly graphical interface
- Hyphenate two or more words modifying a noun as a unit when confusion might result
  - ✓ built-in drive, read-only memory, up-to-date information
- Hyphenate when one word is a past/present participle (-ed/-ing)
  - ✓ left-aligned text, free-flowing form, well-defined schema
- Hyphenate number/letter + noun or participle
  - ✓ two-sided arrow, 5-point star, y-coordinate values

### Suspended compound modifiers

- Avoid suspended compounds (*left- and right-aligned text*) unless space is limited
- If used, include a hyphen with both adjectives: *upper- or lower-right corner*
- Don't suspend one-word adjectives: ✓ *uppercase and lowercase letters*

### Compound nouns

- Hyphenate when one word is abbreviated: *e-book*, *e-commerce*
- Exception: *email* (no hyphen)

### Prefixes

- Generally no hyphen after: auto-, co-, cyber-, giga-, kilo-, mega-, micro-, non-, pre-, re-, sub-, tera-, un-
- Use a hyphen when a confusing word results: *non-native*, *pre-provisioned*
- Use a hyphen to avoid confusion with other words: *co-op*, *re-create*, *re-sign*
- Use a hyphen when stem word begins with a capital letter: *non-XML*
- A prefix modifies a word, not a phrase
  - ✗ non-security related → ✓ unrelated to security
- Double vowel with both pronounced: no hyphen (*reenter*, *cooperate*)

### Capitalization

- Capitalize any part of a hyphenated word that would be capitalized without the hyphen
  - ✓ Customer-friendly, E-Book, Add-ins

## Ellipses

- Don't use an ellipsis (…) except to:
  - Indicate omitted code in technical content
  - Show a pause in conversational UI: ✓ *(Hmm ... looks like that link is broken.)*
  - Indicate omitted text in quotations
- When an ellipsis replaces text mid-sentence, add a space before and after
  - ✓ The quick brown fox … lazy dog.
- When it replaces the end of a quoted sentence, space before, then closing period with no space
  - ✓ The quick brown fox ….
- When UI has an ellipsis, don't include it in instructions
  - ✓ Select **Safety**, and then **Delete browsing history**.
- If no ellipsis character is available, use three periods with no spaces between them

## Exclamation points

- Use sparingly — save them for when they truly count

## Periods

- End all sentences with a period, even two-word sentences
- Use one space after a period, not two
- Don't use periods in headlines, headings, subheadings, UI titles, or simple lists (≤3 words per item)
- In bulleted lists after a colon:
  - Use a period after every item if items complete the introductory phrase
  - No periods if all items are short phrases (≤3 words)
  - Use a period after every item if any item is a complete sentence

## Question marks

- Use questions sparingly — customers want answers, not questions
- Questions are appropriate when the customer needs to make a decision
  - ✓ Do you want to save your changes?

## Quotation marks

- Use double quotation marks (" "), not single, for primary quotations
- Use single quotation marks (' ') for quotes within quotes
- Use straight quotation marks, not curly (Segoe Sans has no curly option)
- Place closing quotation marks outside commas and periods
- Place closing quotation marks inside other punctuation
- Exception: if punctuation is part of the quoted material, place it inside
- Refer to them as *quotation marks* — not *quote marks* or *quotes*
- Don't use quotation marks for emphasis — they're for quotations only

## Semicolons

- Prefer simplifying the sentence (split into multiple sentences or a list) over using a semicolon
- Use between two independent clauses not joined by a conjunction
  - ✓ Select **Options**; then select **Automatic backups**.
- Use between contrasting statements not joined by a conjunction
- Use to separate items in a series that already contains commas — or better, convert to a list

## Slashes

- Differentiate *slash* (/) for URLs from *backslash* (\\) for server/folder paths
- When instructing to type a slash, spell it out first, then symbol in parentheses
  - ✓ Enter two backslashes (\\) ....
- Use a slash to imply a combination: *client/server*, *TCP/IP*, *on/off*
  - Capitalize after the slash if the word before is capitalized: *Country/Region*
- Use two slashes after a protocol: *ftp://example.com*
- Use in server/folder/file paths: *\\\\mslibrary\\catalog\\collect.doc*
- Use between numerator and denominator in inline fractions: *a/x + b/y = 1*
- Don't use a slash as a substitute for *or*
  - ✗ product/service → ✓ product or service
  - Exception: *country/region* and *ZIP Code/postal code* where space is limited
- If UI uses a slash, follow the label but write *or* in body text

## Formatting punctuation

- Format punctuation in the same font style as the main text of the sentence
- If punctuation is part of what the user types, format it like the element
  - ✓ Enter **Balance due:** (colon is bold — user types it)
- If punctuation is not part of the UI element, use main-text formatting
  - ✓ On the **Insert** menu, go to **Pictures**, and then select **From File**.
  - (comma after *Pictures* and period after *File* are not bold)
- Parentheses and brackets: format in main-text style, not the style of their contents
- Match opening and closing parenthesis/bracket style
